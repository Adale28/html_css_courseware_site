Lesson Objectives

This lesson plan includes the objectives, prerequisites, and exclusions of the lesson teaching students how to code with HTML and identify when to use different HTML tags and elements.

Students will be able to:

  • define Hypertext Markup Language (HTML),
  • define HTML tags, elements, and properties,
  • identify common HTML elements,
  • identify different types of properties,
  • Identify the structure of HTML code,
  • Write simple HTML code statements.

Prerequisites

Students should already be familiar with

  • What HTML is used for creating static web page content.

Exclusions

Students will not cover

  • complex HTML elements,
  • The implementation of HTML elements on an editor or IDE only on an online live editor.

Steps to Achieve Objectives:

  • They explore HTML tags and elements through interactive examples.
  • Learners read an overview and watch a short instructional video.
  • They practice with matching activity.
  • They are given sample code and design requirements (e.g., "Create a home page with a header, paragraph").
  • They edit the code live in JSFiddle to meet requirements.
  • Learners submit screenshots and code links.
  • They will share their work via Padlet
  • They reflect on their learning in a Padlet activity.
  • Immediate auto-feedback helps correct errors.

HTML Structure

HTML documents have a specific structure:

This structure includes a <head> section for metadata and a <body> section for content.

HTML Elements

HTML elements are the building blocks of HTML. They consist of tags, attributes, and content. For example: 

    <h1 class="title" id="main-heading">Welcome to HTML!</h1>

In this example, <h1> is the tag, class="title" and id="main-heading" are attributes, and "Welcome to HTML!" is the content.

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the opening tag. For example:

    <a href="https://www.example.com" target="_blank">Visit Example</a>

In this example, href is an attribute that specifies the URL of the link, and target="_blank" opens the link in a new tab.

HTML Comments

Comments are used to leave notes in your code. They are not displayed in the browser. For example:

<!-- This is a comment -->

Comments can help you remember what your code does or provide explanations for others reading your code.

HTML Doctype Declaration

The doctype declaration tells the browser which version of HTML you are using. It should be the first line in your HTML document:

<!DOCTYPE html>

This declaration is essential for ensuring that your HTML is rendered correctly by web browsers.

HTML Best Practices

To write clean and maintainable HTML code, follow these best practices:

  • Use semantic tags (like <header>, <footer>, <article>) to improve accessibility and SEO.
  • Keep your code organized with proper indentation.
  • Use comments to explain complex sections of your code.
  • Validate your HTML using online validators to catch errors.

By following these practices, you can create well-structured and efficient HTML documents.
